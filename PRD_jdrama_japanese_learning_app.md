# PRD: 일본 드라마풍 대사 기반 일본어 학습 앱

## 1) 제품 개요
- 제품명(가칭): `DramaNihongo`
- 한 줄 정의: 일본 드라마의 말투/상황감을 반영한 합법 콘텐츠로 듣기/어휘/문법/쉐도잉을 통합 학습하는 모바일 중심 앱
- 타깃 플랫폼: iOS/Android(우선), Web(관리자/콘텐츠 운영)

## 2) 배경 및 문제 정의
- 기존 일본어 학습 앱은 교과서형 예문 중심이라 실제 회화 톤/속도/감정 표현 학습이 약함
- 학습자들은 `재미`와 `몰입`이 지속 학습의 핵심인데, 현재 학습 도구는 콘텐츠 동기 부여가 낮음
- 드라마풍 대사는 실사용 맥락이 풍부해 표현 습득에 유리하지만, 개인이 합법적으로 쓸 수 있는 형태로 재가공된 서비스가 적음

## 3) 목표
## 제품 목표
- 드라마풍 맥락 학습으로 8주 내 학습 지속률과 회화 체감 향상
- 초중급 학습자 대상 “재미 + 실전성” 결합 학습 경험 제공

## 비즈니스 목표
- 출시 3개월 내 유료 전환율 3~5%
- 월간 리텐션(M2) 25%+

## 4) 핵심 사용자
- 일본어 초중급 학습자 (JLPT N4~N2)
- 드라마/일본 문화 콘텐츠 소비가 많은 20~40대
- 반복 학습 습관이 약해 “짧고 재미있는” 학습 단위를 선호하는 사용자

## 5) 가치 제안 (Value Proposition)
- 드라마풍 실전 대사로 배우는 자연스러운 표현
- 장면 맥락(감정/관계/상황) 기반 기억 강화
- `듣기 → 해석 → 발화(쉐도잉) → 복습` 폐쇄 루프 제공

## 6) 범위
## MVP 포함
- 대사 카드 학습(문장, 후리가나, 번역, 문법 포인트)
- 유튜브 공개 주제 참고 기반 각색 대사 제작 파이프라인
- 구간 반복 재생(A-B loop), 속도 조절(0.75x/1.0x/1.25x)
- 단어장 자동 저장(SRS 복습)
- 쉐도잉 녹음 및 자기 비교(파형/길이/속도 간단 피드백)
- 학습 단계별 파스텔풍 캐릭터 이미지 연동 (Scene/Quiz/Summary)
- 오늘의 학습 루틴(10~15분)
- 학습 진도/연속 학습일(Streak)

## MVP 제외(후속)
- 실시간 발음 채점 고도화(음소 단위)
- 커뮤니티/랭킹
- 드라마 전체 에피소드 스트리밍 기능

## 7) 사용자 시나리오
1. 사용자는 관심 장르(로맨스/의학/법정)를 선택한다.
2. 앱은 레벨별 추천 장면(30~90초)을 제시한다.
3. 사용자는 장면을 보고 핵심 대사 5~10개를 학습한다.
4. 각 대사에서 단어/문법 설명을 확인하고 퀴즈를 푼다.
5. 쉐도잉 후 저장하면 오늘 루틴 완료로 기록된다.
6. 다음 날 SRS 복습 큐로 자동 재노출된다.

## 8) 기능 요구사항
## 콘텐츠/학습
- 대사 메타데이터: 소스ID, 화자, 장면 설명, 난이도, 감정 태그, 라이선스 정보
- 각색 메타데이터: `reference_url`, `topic_id`, `originality_check`
- 문장 분석: 형태소 분해, 후리가나, 어휘 뜻, 문법 포인트
- 퀴즈 유형: 빈칸/순서배열/뜻매칭/청해 선택형
- 학습 세션: 10분 Quick / 20분 Deep 모드

## 플레이어
- 자막 on/off (일본어/한국어)
- 문장 단위 점프
- 북마크 및 어려운 문장 표시
- 캐릭터 이미지 on/off 토글

## 개인화
- 레벨 진단(초기 3분 테스트)
- 오답 기반 추천
- 장르/상황 취향 기반 추천(후속 고도화)

## 9) 비기능 요구사항
- 앱 실행 후 첫 화면 2초 내 로드(캐시 기준)
- 오프라인 캐시: 최근 학습 20개 카드
- 개인정보/학습기록 암호화 저장
- 저작권/라이선스 검증된 콘텐츠만 노출 (원본 페이지 기준 재검증)

## 10) 데이터 및 저작권 정책 (중요)
- 사용 가능 소스:
  - CC/퍼블릭도메인 텍스트 코퍼스
  - 재사용 허용 스톡 영상/오디오
  - 자체 제작(직접 작성/직접 녹음) 대사
- 필수 메타데이터:
  - `license_type`, `source_url`, `author`, `attribution_text`, `commercial_ok`, `derivative_ok`, `valid_from`, `valid_to`
- 정책:
  - 원본 드라마 영상/자막 무단 전재 금지
  - 유튜브/방송 클립은 주제 참고만 허용, 문장 직복제 금지
  - 라이선스 원문 기준으로 상업/2차가공 가능 여부 검증
  - `NC`/`ND` 제약 소스는 배포 정책과 충돌 시 자동 차단
- 사용자 녹음 데이터는 명시적 동의 후 저장/삭제 가능

## 11) 성공 지표 (KPI)
- Activation: 가입 후 24시간 내 첫 학습 완료율
- Engagement: DAU/MAU, 평균 학습 시간, 주간 완료 세션 수
- Learning: 퀴즈 정답률 상승, 복습 완료율, 쉐도잉 완료 수
- Retention: D1/D7/D30
- Revenue: 무료→유료 전환율, 구독 유지율

## 12) 분석 이벤트 설계(요약)
- `lesson_started`, `line_bookmarked`, `quiz_submitted`, `shadowing_recorded`, `srs_review_done`, `subscription_started`
- 각 이벤트 공통 속성: `user_level`, `drama_id`, `line_id`, `session_type`, `duration_sec`

## 13) 기술 아키텍처 (MVP)
- 클라이언트: Flutter 또는 React Native
- API: Python(FastAPI) 또는 Node.js(NestJS)
- DB: PostgreSQL (콘텐츠/유저/학습기록)
- 검색: OpenSearch/pg_trgm (대사 검색)
- 스토리지: S3 호환(오디오/썸네일/자막)
- 어드민 CMS: 콘텐츠 업로드/검수/배포 워크플로
- 라이선스 레지스트리: 자산별 권리 정보/출처/만료 관리
- 콘텐츠 제작 가이드: `docs/content_adaptation_and_visual_style_v1.md`

## 14) 출시 로드맵
## Phase 0 (2주): 검증
- 오픈 라이선스/자체제작 기반 100~200개 대사 카드로 폐쇄 베타
- 학습 루틴/리텐션 가설 검증

## Phase 1 (6~8주): MVP
- 핵심 학습 플로우 + 구독 결제 + 분석 대시보드

## Phase 2 (8주+): 확장
- 추천 고도화, 발음 피드백 고도화, 오픈 콘텐츠 파이프라인 확대

## 15) 리스크 및 대응
- 저작권 리스크: 라이선스 검수 체크리스트 + 배포 전 자동 차단 규칙
- 콘텐츠 운영 비용: 우선 장르/작품 집중으로 ROI 관리
- 학습 피로도: 세션 길이 짧게, 리워드/연속학습 설계

## 16) MVP 완료 기준 (Definition of Done)
- 신규 사용자가 15분 내 첫 학습 완료 가능
- 대사 카드 학습/퀴즈/SRS/쉐도잉 흐름이 오류 없이 연결
- 핵심 KPI 이벤트가 누락 없이 수집
- 출처/라이선스 검증 통과한 콘텐츠만 프로덕션 배포
