## 1) 목적
MVP에서 사용할 일본어 학습용 대사/오디오/썸네일의 법적 사용 가능 범위를 명시하고, 배포 차단 조건을 정의한다.

## 2) 자산 유형별 필요 권리
- 텍스트(학습 대사): 복제/전송/2차가공 허용 여부
- 오디오(학습 대사 음원): 복제/전송/편집 허용 여부
- 이미지(장면 분위기 썸네일): 복제/전송/상업 사용 허용 여부
- 메타데이터(출처/저작자/라이선스): 표시 의무(Attribution) 충족 여부

## 3) MVP 허용 범위 (초안)
- 콘텐츠 수량: 최소 100개 대사 카드
- 클립 길이: 카드당 최대 3~10초 구간
- 용도: 학습 목적 앱 내 재생/반복/퀴즈
- 지역: 대한민국 우선 출시
- 기간: 라이선스 유효기간 내 서비스, 만료 시 즉시 비노출 가능해야 함

## 4) 금지/제한 항목
- 비공식 자막, 커뮤니티 전사본, 무허가 팬번역 사용 금지
- 원본 드라마 영상/대사의 무단 전재 금지
- 라이선스 범위를 벗어난 국가 노출 금지
- 라이선스 범위를 벗어난 마케팅 채널 2차 활용 금지
- 만료 자산의 캐시 잔존 노출 금지
- `ND`(변형 금지), `NC`(비상업) 제약과 서비스 모델 충돌 시 사용 금지

## 5) 소스 체크리스트 (배포 전 필수)
- [ ] 원본 페이지 URL 확인
- [ ] 라이선스 원문/약관 링크 확보
- [ ] 상업 이용 가능 여부 확인
- [ ] 2차가공 가능 여부 확인
- [ ] Attribution 문구 확보
- [ ] 국가/기간/매체(모바일/웹) 범위 확인
- [ ] 만료/철회 시 삭제 정책 확인

## 7) 시스템 요구사항 (법무 반영)
- 자산별 `license_id`, `territory`, `valid_from`, `valid_to` 메타데이터 필드 필수
- 앱/서버에서 `valid_to` 경과 시 자동 비노출 처리
- CDN 캐시 purge 자동화 훅 필요
- 운영자 CMS에서 자산 상태(`active/hold/expired`) 강제 전환 가능해야 함

## 8) 리스크 및 대응
- 리스크: 소스별 라이선스 해석 불일치
  - 대응: 원본 페이지 스냅샷 + 검수자 코멘트 기록
- 리스크: 라이선스 만료 후 잔존 노출
  - 대응: 만료 D-7 경고 + D-day 자동 차단 + 로그 보존
- 리스크: 플랫폼 약관 위반 소스 혼입
  - 대응: 승인 전 소스 URL 화이트리스트 검증
- 리스크: 지역 제한 위반
  - 대응: 계정 국가/접속 IP 기반 접근 제어
